VS Code使用（C/C++）

这几天由于复习C/C++的需要，了解学习了一下VS Code这个目前十分受欢迎的编辑器。作为VS Code的初级玩家，有必要写一遍小总结，记录在学习过程中学到一些命令行、程序编译和调试，以及使用VS Code的知识。

准备工作

  • VS Code安装

    前往微软官网下载并安装好visual stdio code，安装步骤这里不再细讲，不会的可以自行度娘或者知乎。

  • C/C++编译环境

    与其他IDE不同（如VS studio）VS Code只是个纯文本编译器，没有自带有C/C++的编译器和其他功能，所以我们需要下载安装C/C++编译器，并配置相应的环境变量

    • WinGM安装

      MinGW是gcc在Windows下的移植，gcc是世界上最流行的C/C++编译器组合。但gcc这个名字也指编译C语言的那个程序，g++才是C++编译器。本人使用的是MinGM-w64。将下载得到的文件压缩包进行解压，建议放到一个独立的且不容易被删掉的地方。

    • 配置环境变量

      这里注意一下WinGM解压后的bin文件夹的完整路径，把它添加到环境变量的PATH中。实在不会配置环境变量的，建议度娘学习下。

    • 验证MinGM是否可用

      打开cmd 输入gcc -v出现版本信息，说明MinGM可用，如果未出现相关信息，需要重新配置。注意，这里非常关键。

  • 扩展

    打开VS code在扩展商店extensions中搜索下载即可

    • C/C++，又名 cpptools，提供Debug和Format功能
    • chinese(simplified)，中文语言包
    • Code runner，右键即可编译运行单文件，很方便；唯一缺点就是无法Dubug

  • 配置几个.json文件(C和C++要分别配置，新建并放在.vscode文件下)

    创建一个你打算存放代码的文件夹，称作工作区文件夹；注意:路径不能含有中文和引号，最好不要有空格。C和C++需要分别建立不同的文件夹,也不要选上一段建出来的文件夹，源代码和编译器要分开放

    打开VS Code，选打开文件夹；后点新建文件夹，名称为.vscode。在该文件夹下创建 launch.json，tasks.json，settings.json。这几个文件的内容如下,复制粘贴即可.

    • launch.json

    • settings.json

    • tasks.json

    • c_cpp_properties.json（可有可无）

编译和运行

  • 在对应的工作区下新建一个文件夹（可以不在.vscode下）用来存放编写的源代码

  • 终端命令(直接在终端上用命令行编译和运行)

    • 编译(注意C和C++使用的是不同的编译器)

      C gcc 文件名.c -o 文件名.exe

      C++ g++ 文件名.cpp -o 文件名.exe

      注意:C和C++使用的是不同的编译器

    • 运行 ./ 文件名.exe
